# Mini Shop — 管理后台前端文档

## 入口

- 页面：`public/admin.html`
- 逻辑：`public/admin.js`
- 权限：仅管理员角色可访问（`checkAuth()` 校验）

## 布局结构

左侧边栏 + 右侧内容区：

```
┌──────────────┬───────────────────────────┐
│  Mini Shop   │                           │
│  📦 商品管理  │      管理内容区域          │
│  📋 订单管理  │                           │
│  👥 用户管理  │                           │
│  🛒 回到商城  │                           │
└──────────────┴───────────────────────────┘
```

三个管理区域通过侧边栏按钮切换，同一时间只显示一个。

## 功能模块

### 商品管理

| 功能 | 操作 |
|------|------|
| 商品列表 | `GET /api/products` → 渲染表格 |
| 添加商品 | 点击「添加商品」按钮 → 填写表单 → `POST /api/admin/products` |
| 编辑商品 | 点击「编辑」→ 加载商品数据到表单 → `PUT /api/admin/products/:id` |
| 删除商品 | 确认弹窗 → `DELETE /api/admin/products/:id` |

**表单字段**：商品名称（必填）、价格（必填）、分类（必填）、描述、图片URL（留空自动生成随机图片）

### 订单管理

| 功能 | 操作 |
|------|------|
| 订单列表 | `GET /api/orders`（管理员可见全部）→ 渲染表格 |
| 订单详情 | 点击「详情」→ 弹窗展示收货人、手机、地址、商品列表、合计 |
| 状态更新 | 下拉选择状态 → `PUT /api/admin/orders/:id/status` |

**订单状态流转**：待处理 → 已付款 → 已发货 → 已完成（或 → 已取消）

### 用户管理

| 功能 | 操作 |
|------|------|
| 用户列表 | `GET /api/admin/users` → 渲染表格 |
| 角色切换 | 点击「设为管理员/设为用户」→ `PUT /api/admin/users/:id/role` |
| 删除用户 | 确认弹窗 → `DELETE /api/admin/users/:id`（不能删除自己） |

## XSS 防护

与商城前端一致，所有动态内容通过 `esc()` 函数转义 HTML 特殊字符。

## Toast 通知

复用同一套 `showToast(msg, type)` 函数，操作成功/失败时给出即时反馈。
